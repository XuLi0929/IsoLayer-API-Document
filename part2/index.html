Migrating to IsoLayer v2 v2.4.0

Step 2 of 2 — Execute the Migration

With the cleanup script prepared, you can now run the migration. This will remove the legacy state tree and allow the IsoLayer v2 SDK to initialize cleanly.

Run the cleanup script

# Execute the legacy state cleanup
bash cleanup.sh

Verify and install IsoLayer v2

After the script completes, confirm the legacy path has been cleared and then install the new SDK:

# Verify cleanup was successful
ls /root/
# Expected: empty or "No such file or directory"

# Install IsoLayer v2
pip install isolayer==2.4.0
Note: If ls /root/ still shows files, the cleanup script may not have had sufficient permissions. Re-run with sudo bash cleanup.sh.
Migration complete!

Your system is now ready for IsoLayer v2. The new SDK will store all state data in /opt/isolayer/ going forward.
